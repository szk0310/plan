## 0. 本提案の前提

前回提案（今月初）に対し、大林役員より以下のご指摘をいただきました。

> 「うちはデザインが長所なのに、AIに傾きすぎている」

**本提案はこの批判を正面から受け止め、再設計しました。**
結論：「デザイン」と「AI」は対立しない。むしろ、shiro にしかできない統合の仕方があります。

## 1. 現状認識

### 業界で何が起きているか

Webの役割が変わっています。

| 旧来のWeb | これからのWeb |
|---|---|
| 会社情報を載せる「看板」 | 顧客と対話する「営業担当」 |
| 全員に同じページを見せる | その人の文脈に合わせて変わる |
| 制作して終わり | 訪問者データを蓄積して進化する |

この変化は、**デザインの終わりではなく、デザインの役割の拡張**です。
「何を作るか」だけでなく「誰に、いつ、どう届けるか」がデザインになりました。

> **研究的裏付け：**
> SME AI Research（2025）の論文 *"Generative AI Affordance in UX Design"* は、
> 生成AI時代のデザイン原則として「操作手順の提示」から**「目標達成の支援」**へのパラダイムシフトを提唱しています。
> ユーザーは「何をすればいいか」ではなく「何を達成できるか」を直感的に感じ取れる設計が必要とされており、
> これはまさに本提案のウィザード設計（5問で課題を整理 → 構成案を提示）と合致します。
> — 出典：https://research.smeai.org/generative-ai-affordance-ux-design/

### shiro の現在地

- 強み：繊細なデザイン × 正確なコーディング × 自社開発システム
- 課題：HPがその強みを体現できていない
- 機会：この変化の時代に、制作会社として最も説得力のある立場にいる

## 2. リニューアルコンセプト

### コンセプトワード

```
届くデザイン、をつくる。
```

### なぜこの言葉か

shiro の既存タグライン "Engage Person to Company via web" を深化させた言葉です。

```
美しいデザインをつくる  ≠  届いた

届く  ＝  正しい相手に・正しいタイミングで・その人だけの言葉で
```

「届くデザイン」には三層の意味があります：

| 層 | 意味 | shiro の根拠 |
|---|---|---|
| ① 造形的に届く | 見た瞬間に伝わる美しさ | 芸大出身の繊細なデザイン力 |
| ② 戦略的に届く | 正しい相手に、正しいメッセージを | 自社開発システムの活用 |
| ③ 個人的に届く | その一人だけの体験として | 新しい技術の活用 |

**AIという言葉は一切使わずに、同じ未来を語ることができます。**

## 3. HPリニューアルの骨子

### 3-1. HPの目的を再定義する

```
旧：会社紹介 + 実績掲載 + 問い合わせ受付

新：shiro の「届くデザイン」思想を、体験させる場
```

訪問者が「このHPを使ったら、自分の課題が整理できた」と感じて帰る。
その体験が、そのままshiroへの信頼になります。

### 3-2. HP自体が「デモ」になる

提案の核心です。

> **shiroのHPは「届くデザイン」の実物見本です。**

新規訪問者は、5つの問いに答えるだけで、
自社のLPやサイトの構成案が目の前に現れます。

```
Q1. 誰に届けたいですか？
Q2. 今の一番の課題は？
Q3. 商材・サービスの性質は？
Q4. 理想の印象は？
Q5. 今のサイトの状態は？
    ↓
あなたの会社に最適なLP構成案が表示されます
```

これは「AI機能を載せた」のではありません。
**「デザインの届け方を、体験として設計した」**のです。

### 3-3. HPが営業を自動化する

体験した訪問者が「相談したい」と思ったとき、メールアドレスを入力します。
その瞬間から、以下が自動で動きます：

```
訪問者が「相談する」をクリック
         ↓
顧客情報が社内システムに登録
         ↓
担当者のSlackに即時通知
         ↓
翌日、文脈を踏まえたフォローアップが自動送信
         ↓
商談へ
```

重要なのは、**この仕組み自体がshiroの新サービスである**ということです。

## 4. ビジネス上の意義

### 4-1. shiroの売るものが増える

```
旧：Webサイト・LPを制作します
新：「届くデザイン」の仕組みを、まるごと提案します
```

**制作単価の競争から抜け出し**、「制作 + 仕組み」のパッケージ提案が可能になります。

### 4-2. 自社HPが最強の営業ツールになる

「うちのHPを見てください。来た方の課題に合わせて、構成案が出てきます。
問い合わせいただいたら、翌日にはフォローのご連絡が届きます。
これを御社でも、という提案です。」

**製品デモが不要になります。HPがデモです。**

### 4-3. 競合との明確な差別化

| 競合他社 | shiro（提案後） |
|---|---|
| 実績ポートフォリオを並べる | HPで体験させる |
| 「AIを活用します」と説明する | 黙って体験させる |
| 制作会社として提案する | 「届く仕組み」ごと提案する |

## 5. デザイン方針（大林役員へ）

前回提案との最大の違いはここです。

### NG（前回）
- 暗背景 + ネオンカラー + テック系フォント
- 「AI企業っぽさ」が全面に出ていた
- shiroらしくなかった

### OK（今回）
- **白ベース・余白重視・タイポグラフィが主役**
- デザイン会社の矜持を前面に
- 技術は「見えない」ところで働く

```
カラー：ほぼ白 + ウォームグレー + アクセント1色
フォント：和文は繊細なセリフ系、欧文は洗練されたサンセリフ
動き：必然性のあるものだけ。派手さより精度
温度感：知性的だが冷たくない
```

**「AIっぽいHP」ではなく、「shiroらしいHP」が、結果として最先端である。**
これが今回のデザイン戦略です。

## 6. 実装について

### 既存資産の最大活用

新規開発が必要なのは **フロントエンドのみ** です。

| 機能 | 状態 |
|---|---|
| 顧客情報管理（CRM） | 稼働中 |
| Slack通知 | 稼働中 |
| AI自動フォローアップ | 稼働中 |
| HP フロントエンド | 新規作成 |

すでに自社で構築・運用している基盤の上に、HPを接続するだけです。

### スケジュール感

```
Week 1-2：デザインカンプ作成・社内レビュー
Week 3-4：フロントエンド実装
Week 5  ：既存システムとの接続・テスト
Week 6  ：公開
```
