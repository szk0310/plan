# TASK_12: Slack /crm-list 強化 — 確率ベース表示 + ページネーション

> **目的**: Slack を CRM の「メイン操作画面」として完成させる
> **優先度**: 🟡 高
> **依存**: TASK_10（確率エンジン）

---

## 1. 設計方針

Slack は「入力 & アクション」のメイン UI。
日常業務の 90% は Slack 内で完結させる。

### 現状の問題

| 問題 | 影響 |
|------|------|
| 20件固定で全件表示 | 21件目以降が見えない |
| deals に旧ステージ（📋初期 / ✅適格...）表示 | 確率一本化と矛盾 |
| contacts に lifecycle_stage がない | Web と表示が不整合 |
| ネクストアクションが一覧に出ない | Slack から遷移しないと見えない |
| 編集・削除が Slack からできない | Web に行く必要がある |

## 2. /crm-list contacts の改善

### 2.1 表示の変更

```
旧:
  🧊 *田中太郎*（ABC株式会社） | 案件化: 45%
  [詳細]

新:
  🌤️ *田中太郎*（ABC株式会社）
  見込客 | 案件化: 45% | 最終: 3/15
  💡 フォローアップ電話を推奨
  [詳細] [メモ追加] [編集]
```

変更点:
- lifecycle_stage のラベル追加（見込客 / 商談中 / 既存顧客 / 休眠）
- 最終接触日の表示
- AI 推奨アクション（1行）
- 「メモ追加」「編集」ボタン追加

### 2.2 ページネーション

```
顧客（23件中 1-10件）

  ... 10件の一覧 ...

  [◀ 前の10件] [次の10件 ▶]
```

実装:
- `offset` パラメータでページ管理
- ボタンの `value` に `contacts:keyword:offset` を JSON エンコード
- `action_id: 'crm_list_paginate'` でページ遷移

### 2.3 フィルタ

```
/crm-list contacts          → 全件
/crm-list contacts hot      → nurturing_stage = hot のみ
/crm-list contacts 見込み    → lifecycle_stage = prospect のみ
/crm-list contacts 田中     → キーワード検索（既存）
```

## 7. 共通ラベル定数の作成

Web と Slack で同じラベルを使うために:

```typescript
// api/src/config/crm-labels.ts（新規）

export const LIFECYCLE_LABELS: Record<string, string> = {
  prospect: '見込客',
  active: '商談中',
  customer: '既存顧客',
  inactive: '休眠',
  churned: '離反',
};

export const NURTURING_EMOJI: Record<string, string> = {
  cold: '🧊',
  warm: '🌤️',
  hot: '🔥',
};

export const PROBABILITY_EMOJI = (prob: number): string => {
  if (prob >= 91) return '⭐';
  if (prob >= 61) return '🔥';
  if (prob >= 31) return '🌤️';
  return '🧊';
};

export const PROBABILITY_BAND = (prob: number): string => {
  if (prob >= 91) return '確実';
  if (prob >= 61) return '高';
  if (prob >= 31) return '中';
  return '低';
};
```
