# TASK_11: Web をダッシュボード特化にリファクタ

> **目的**: Web の役割を「閲覧・分析・設定」に絞り、CRUD は Slack に寄せる
> **優先度**: 🟡 高
> **依存**: TASK_10（確率エンジン刷新）

---

## 1. 設計方針

### Web の役割（やること）

| 用途 | 画面 |
|------|------|
| パイプライン俯瞰 | DealBoard — 確率帯カンバン + 金額サマリー |
| 確率推移の可視化 | DealDetail — 折れ線グラフ（deal_progress_history） |
| KPI ダッシュボード | Dashboard — 受注確率分布、活動量、センチメント傾向 |
| 経営レポート | Dashboard — 今月パイプライン総額、受注見込み |
| テナント設定 | Settings（SaaS 化時に必要） |

### Web の役割（やらないこと → Slack へ）

| 操作 | 理由 |
|------|------|
| コンタクトの作成・編集・削除 | Slack で自然言語 or /crm で済む |
| ノート追加 | Slack メッセージで済む |
| 商談のステータス変更 | AI が自動判定する |
| ナーチャリング承認 | Slack ボタンで済む |

## 3. グラフライブラリ

### 選定: Chart.js（軽量、Vue 3 対応）

```bash
npm install chart.js vue-chartjs
```

理由:
- バンドルサイズが小さい（tree-shakeable）
- 折れ線・棒・レーダーチャートを全てカバー
- Vue 3 の Composition API と相性が良い
- 他の選択肢（D3.js は過剰、Recharts は React 用）

## 4. API エンドポイント追加

| エンドポイント | 用途 |
|---------------|------|
| `GET /api/dashboard/sentiment-summary` | 今週のセンチメント集計 |
| `GET /api/dashboard/probability-distribution` | 確率帯ごとの件数 + 金額 |
| `GET /api/dashboard/recommended-actions` | AI 推奨アクション TOP N |

既存エンドポイントの変更:
- `GET /api/deals` → `stage` フィルタを廃止、確率帯フィルタに変更
- `GET /api/deals/:id/progress-history` → そのまま活用（グラフ用）
