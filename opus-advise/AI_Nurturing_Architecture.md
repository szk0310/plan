# AIナーチャリング：設計と戦略の提言

## ビジョン
> 営業マンが寝ている間にも、AIが顧客との関係を温め、
> 朝起きたら「〇〇さんからアポ取れました」とSlackに届いている世界。

CRM部分は徹底的にシンプルに。**AIナーチャリングこそが SlackSFA の存在意義。**

---

## 1. AIナーチャリングとは何か（定義）

従来の「マーケティングオートメーション（MA）」との決定的な違い：

| | 従来のMA（Salesforce等） | SlackSFA のAIナーチャリング |
|:---|:---|:---|
| **やること** | シナリオ通りにメールを送る | **相手の反応を見て、AIが次の行動を考える** |
| **設定コスト** | シナリオ設計に数週間 | **営業方針を伝えるだけ（プロンプト）** |
| **パーソナライズ** | 名前の差し込み程度 | **相手の状況に合わせた完全個別対応** |
| **人間の関与** | 設定後は放置 | **重要な判断だけ人間が承認** |

**一言で言えば：「優秀な営業アシスタントを、全顧客に1人ずつ付ける」こと。**

## 3. AIが「嘘をつかない」ためのガードレール設計

### 3-1. RAG（Retrieval-Augmented Generation）の実装

AIが参照できる情報を **「自社が公式に出している情報」だけ** に限定します。

```
[参照可能な情報源]
├── 自社サービス資料（PDF/Google Docs）
├── 料金表
├── FAQ・よくある質問
├── 過去の成功事例（匿名化済み）
└── 営業方針ドキュメント（szk様が定義）

[参照不可（AIが勝手に作ってはいけない情報）]
├── 価格の値引き・特別条件
├── 納期の確約
├── 競合他社の批判
└── 技術的な保証・SLA
```

### 3-2. 営業方針プロンプト（szk様が定義するもの）

```
あなたは〇〇株式会社の営業アシスタントです。

【行動原則】
- 課題解決型の提案を心がける（押し売りしない）
- 相手の予算やタイミングを尊重する
- 「わからないこと」は正直に「担当に確認します」と答える
- 競合を悪く言わない

【禁止事項】
- 価格の交渉や値引きの約束
- 納期の確約
- 契約条件の提示
```

このプロンプトは **テナントごと（顧客企業ごと）にカスタマイズ可能**にします。
SaaS化した時、各企業が自社の営業スタイルを AI に教えられる仕組みです。

## 4. 承認フロー（Human-in-the-Loop）の詳細

### Slackでの体験イメージ

```
🤖 AIナーチャリング Bot
━━━━━━━━━━━━━━━━━━━━━━━

📩 田中太郎さん（ABC株式会社）への返信案

【きっかけ】
田中さんから資料請求のメールが届きました（3分前）

【AIの判断】
関心度: ⭐⭐⭐⭐☆ (高い)
理由: 2回目の問い合わせ。前回は予算未定で見送り。
      今回は具体的な機能について質問 → 予算確保済みの可能性

【下書き】
━━━━━━━━━━━━━━━━━━━━━━━
田中様

お世話になっております。
先日はお忙しい中お問い合わせいただき
ありがとうございます。

ご質問いただいた〇〇機能について、
詳しい資料をお送りいたします。

もしよろしければ、15分ほどオンラインで
ご説明させていただくことも可能です。
ご都合の良い日時をお知らせいただけますか。
━━━━━━━━━━━━━━━━━━━━━━━

[👍 このまま送信]  [✍️ 修正して送信]  [❌ 送らない]
[📞 自分で電話する]  [⏰ 明日送信]
```

### ポイント
- **AIが判断理由を明示する**（なぜこの人に今連絡すべきか）
- **担当者は「承認」するだけ**（文章を書く必要がない）
- **「自分で電話する」選択肢**で、AIから人間へのスムーズなバトンタッチ

## 5. 段階的な実装ロードマップ

### Phase A：受動型（まず聞くだけ）
- メールや問い合わせの受信を `activity_logs` に蓄積
- AIが「この人、動きがありました」とSlackに通知するだけ
- **人間が全てを判断する**
- 🎯 目的：データの蓄積と、AI通知の精度検証

### Phase B：提案型（下書きを作る）
- AIが返信の下書きを生成し、Slackに提示
- 承認フロー（👍/✍️/❌）を実装
- **人間が承認しないと送信されない**
- 🎯 目的：AIの文章品質と判断精度の検証

### Phase C：自律型（信頼できたら任せる）
- Confidence Score 90以上 → 自動送信（ただし事後通知）
- Confidence Score 70-89 → 承認フロー
- Confidence Score 70未満 → 人間にバトンタッチ
- 🎯 目的：営業の完全自動化

**Phase A から始めれば、リスクゼロで AI ナーチャリングのデータ基盤を作れます。**

## 6. コスト構造（AIナーチャリング部分）

| 処理 | モデル | 1回あたりコスト | 月間想定回数 | 月額 |
|:---|:---|:---|:---|:---|
| 状況分析 | Flash | ~0.1円 | 1,000回 | ~100円 |
| 行動決定 | Pro | ~3円 | 200回 | ~600円 |
| 文章生成 | Sonnet | ~5円 | 100回 | ~500円 |
| **合計** | | | | **~1,200円/月** |

**月額3,000〜5,000円のSaaS → 利益率60〜75%**

## まとめ：CRMはシンプルに、AIナーチャリングに全力

| 領域 | 方針 |
|:---|:---|
| **CRM（土台）** | Contact一本化。入力はSlackで30秒。余計な機能は作らない |
| **AI ナーチャリング（核）** | 3層モデル（分析→判断→生成）で品質とコストを両立 |
| **安全性** | RAG + 承認フロー。AIが暴走しない設計を最初から組み込む |
| **段階的リリース** | Phase A（通知のみ）→ B（下書き）→ C（自律）で信頼を積み上げる |

> **このプロダクトの売り文句：**
> 「CRMの入力は Slack で30秒。あとは AI が関係を温めます。」
