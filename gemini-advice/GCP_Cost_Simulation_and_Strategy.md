# SlackSFA：GCP 月額コスト試算と AI モデル戦略

商用化における AI モデルの API コストは「変動費」となり、利益率に直結します。
どの処理にどのモデルを割り当てるべきか、ビジネス視点で試算しました。

## 2. 業務フロー別のモデル使い分け案（コスト最適化）

| プロセス | 推奨モデル | 理由 |
| :--- | :--- | :--- |
| **音声メモの全文要約** | **Gemini 1.5 Flash** | 文脈は単純。圧倒的に安く、長文処理に強い。 |
| **リード属性の自動抽出** | **Gemini 1.5 Flash** | 名前、会社名、電話番号の抽出は Flash で十分。 |
| **AI 案件化確率 (Scoring)** | **Gemini 1.5 Pro** | 「行間を読み取る」力が必要。案件化の根拠は高品質であるべき。 |
| **顧客への返信下書き作成** | **Claude 3.5 Sonnet** | 日本語の自然さ、営業マンらしい洗練された文章作成に長けている。 |

## 3. 月間コスト・シミュレーション（1 営業マンあたり）

**前提条件:**
- 1日 10 件のリード登録、20 件のメモ入力。
- 合計月間 600 回の AI 処理。

### A. 全てを Gemini 1.5 Pro で行った場合
- 月間コスト：約 **1,500円 〜 2,500円**
- 懸念点：利益率を圧迫する。

### B. 戦略的使い分け（Flash 8割 : Pro 2割）を行った場合
- 月間コスト：約 **300円 〜 600円**
- メリット：月額 3,000円〜5,000円の SaaS モデルでも **利益率 80% 以上** を確保可能。

---

## 4. ビジネス的な懸念点への仕様提言

### ① コスト予測・制限機能
- **仕様:** 顧客（テナント）ごとに「月間 AI 利用上限」を設定できる管理機能。
- **価値:** 運営側が API コストで赤字になるリスクをゼロにします。

### ② モデルの動的切り替え（Model Routing）
- **仕様:** メモの文字数や重要度に応じて AI が「これは Flash でいい」「これは Sonnet で書くべき」と自己判断して API を呼び分ける。
- **価値:** 最高のクオリティと最低のコストを両立します。

### ③ キャッシュの活用
- **仕様:** 同じ顧客情報の再解析など、前回の結果と変わらない場合は DB の `ai_score` を再利用し、API を叩かない。
